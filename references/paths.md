# Paths, Output Format, and Obsidian Write Conventions

## Workspace Directory

Scripts place cookies, browser user data, and default article output under the directory specified by the **`OPENCLAW_WORKSPACE`** environment variable; when unset, this defaults to **`~/.openclaw/workspace/`**. The placeholder `{workspace}` is used throughout.

### Persistent Files

| Purpose | Path |
|---------|------|
| Cookie | `{workspace}/zhihu_cookies.json` |
| Playwright user data | `{workspace}/chrome_user_data/` |
| Default article directory | `{workspace}/zhihu_articles_{collectionId}/` |
| Default image directory | `{article-output-dir}/images/` |

## Batch-Fetch Command Format

```bash
python fetch_zhihu_batch.py <list-file> [output-dir] [image-dir]
```

| Argument | Description |
|----------|-------------|
| **list-file** | JSON produced by `fetch_zhihu_collection.py` / `fetch_zhihu_history.py` |
| **output-dir** | Optional; defaults to `{workspace}/zhihu_articles_{collectionId}/` (`collectionId` is inferred from the list filename) |
| **image-dir** | Optional; defaults to `{output-dir}/images/`; only needed when specifying a custom image directory |

## Directory Structure Example

```
zhihu_articles_{collectionId}/
├── _progress.json          # resumable-fetch checkpoint
├── images/                 # default image directory
│   └── ...
├── 0001_article-title.md
└── ...
```

## Single-Article Format

- YAML frontmatter: `title`, `author`, `source`, `url`, `voteup`, `images`, etc.
- Body is Markdown; image references point to filenames under the local `images/` directory (or relative paths generated by the script).

```markdown
---
title: "Article Title"
author: "Author Name"
source: zhihu
url: "https://..."
voteup: 123
images: 5
---

# Article Title

> Author: xxx | Original: [Zhihu link](https://...)

Body text...
```

Notes fetched via the personal-history flow carry additional interaction metadata:

```yaml
interaction_action: "赞同了回答 (liked answer)"
interaction_time: 2026-03-20T10:17:57.235000+00:00
interaction_date: 2026-03-20
tags: [zhihu, programming-and-development, 赞同了回答]
```

## Obsidian Write Conventions

### Configurable Output Names

| Setting | Default | Override (flag) | Override (env var) |
|---------|---------|-----------------|---------------------|
| Root folder inside Vault | `Zhihu Collection` | `--root-folder` | `ZHIHU_OBSIDIAN_ROOT` |
| Failure-list filename | `fetch-failures.md` | `--failures-name` | `ZHIHU_FAILURES_FILE` |

**Resolution order (highest to lowest priority):**

1. CLI flags `--root-folder` / `--failures-name`
2. Environment variables `ZHIHU_OBSIDIAN_ROOT` / `ZHIHU_FAILURES_FILE`
3. English defaults: `Zhihu Collection` / `fetch-failures.md`

> **Migration note for existing Chinese vaults:** If your vault already uses the old Chinese layout (`知乎收藏/` folder and `抓取失败.md` failure list), either rename the folder and file to the new defaults, **or** set the environment variables to keep the old layout:
>
> ```bash
> export ZHIHU_OBSIDIAN_ROOT=知乎收藏
> export ZHIHU_FAILURES_FILE=抓取失败.md
> ```

### Vault Path Resolution (`write_to_obsidian.py`)

1. **Command-line argument** (the user supplies the Vault root path directly)
2. **`OBSIDIAN_VAULT`** environment variable (single path), if no argument was given
3. Script scans common directories; prompts interactively when multiple matches are found

### Classification and Write Path (`write_to_obsidian.py`)

- **Classification:** prefers aligning with existing subdirectories under `Zhihu Collection/`; otherwise classifies by content keywords; falls back to "Uncategorized"
- **Write path:** `{Vault}/Zhihu Collection/{category}/{article-title}.md`; image-sync rules are in the script (the target Vault typically has a shared `images/` directory)
- Source side: looks for `<article-dir>/images` first, then falls back to sibling `zhihu_images/`

### Personal-History Write (`write_zhihu_history_to_obsidian.py`)

- Writes to `{Vault}/Zhihu Collection/{category}/` by default
- Scans existing notes for `url` frontmatter and **deduplicates by URL** before writing, preventing duplicate imports
- Preserves interaction time and activity-label tags
